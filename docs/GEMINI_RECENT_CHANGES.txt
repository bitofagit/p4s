1. TL;DR (current state)
================================================================================

- Godot 4.6, GL Compatibility renderer, 60 fps cap, 30 ms audio latency.
- Flora uses a pre-baked 14×15 grid atlas (flora_atlas.png) on three TileMapLayers:
  Canopy (z=1), Understory (z=2), Ground (z=3). Y-sort is OFF on all flora layers
  (y-sort was ~1 canvas item per planted cell → thousands of draws on VMware).
- When zoomed out, full-res flora layers hide and **low-res LOD atlas TileMaps** show
  instead (NOT coloured rectangles anymore — old FloraLodOverlay was removed Jul 14).
- VMware / virtual GPU / old iGPU builds auto-enable "potato mode" via
  MetaManager.low_end_gpu (terrain streaming, fewer particles, zoom limits, etc.).
- Windows exe export was fixed: raw CSV/JSON now pack correctly (seed inventory was
  empty before because Godot imported CSVs as translations instead of shipping them).

2. FLORA LOD ATLAS (Jul 14 — latest)
================================================================================

PROBLEM
  Sprites disappeared when zoomed out (Mac editor + VMware). Earlier fix used flat
  coloured rectangles (FloraLodOverlay) — user wanted actual low-res plant sprites.

SOLUTION
  Offline baker downscales the full flora atlas into two PNG tiers in:
    assets/base/sprites/atlas/lod/
      flora_atlas_lod_mid.png   — 50% sprite centred in each 200 px cell
      flora_atlas_lod_far.png   — 25% sprite centred in each 200 px cell
  Zoom thresholds in data/flora_lod_map.json:
    Full atlas:  zoom >= 0.38
    Mid LOD:     0.20 <= zoom < 0.38   (default farm zoom ~0.25 → mid LOD)
    Far LOD:     zoom < 0.20

RUNTIME
  FloraLodLayers (scripts/flora_lod_layers.gd) — Node2D with mid/far TileMapLayers
  per flora layer (canopy / understory / ground). Same grid coords as full atlas.
  world.gd _update_zoom_culling() toggles full layers vs LOD tier via
  PlantGrowth.flora_lod_tier_for_zoom(zoom).
  starting_map.gd paints LOD cells in the same path as full flora (_paint_flora_cell).

BAKE COMMANDS
  # After flora_atlas.png exists (run bake_flora_atlas.gd first if needed):
  godot --headless --path . --script res://scripts/tools/run_flora_lod_bake.gd
  godot --headless --path . --import

  Editor alternative: File → Run on scripts/tools/bake_flora_lod_atlas.gd

KEY FILES
  scripts/tools/flora_lod_atlas_baker.gd   — baker logic
  scripts/tools/run_flora_lod_bake.gd      — headless CLI wrapper
  scripts/flora_lod_layers.gd              — runtime LOD TileMapLayers
  scripts/plant_growth.gd                  — flora_lod_meta(), flora_lod_tier_for_zoom()
  data/flora_lod_map.json                    — tier paths + zoom thresholds
  DELETED: scripts/flora_lod_overlay.gd     — old coloured-rect stand-in

3. VMWARE / LOW-END GPU PERF (Jul 10)
================================================================================

MetaManager.low_end_gpu auto-detects VMware SVGA, virtual GPUs, and (on p4s-5 build)
old Intel HD / early UHD / AMD APU names.

When low_end_gpu is true:
  - Terrain TileMap streaming: only paint cells near camera (not full 10k map)
  - Flora y-sort disabled; rendering_quadrant_size = 32
  - Rain particles reduced (500 → 80)
  - Structure overlay viewport-culled; river/swale washes skipped on low-end
  - Capillary/triage redraws throttled; fence overlay skipped
  - Camera start zoom / min zoom raised so player can't zoom out to whole farm
  - dynamic_zoom_culling swaps full flora ↔ LOD tiers (never "nothing")

Graphics Settings panel (ui_graphics_panel.gd) — user toggles:
  - render_groundcover (default OFF)
  - render_understory (default ON)
  - dynamic_zoom_culling (default ON)

Perf logging example:
  [P4S perf] fps=55 avg=18ms draws=400 objs=900 ...
  [P4S spike] frame=133ms (process=0ms ...) when hitches occur

5. FLORA ATLAS / LAYER STACK (context for design chats)
================================================================================

Pre-baked atlas replaces runtime PNG stitching (removed for perf).
Baker: scripts/tools/flora_atlas_baker.gd → flora_atlas.png + data/flora_atlas_map.json

Layer offsets (world pixels):
  canopy:     (-25, -35)
  understory: (0, 0)  [check starting_map.gd FLORA_LAYER_OFFSETS]
  ground:     per FLORA_LAYER_OFFSETS

Growth stages 0–5 map to atlas grid coords via flora_atlas_map.json tiles[plant_id][stage].

PlantGrowth (class_name) is the static API for atlas coords, LOD meta, growth stages.
