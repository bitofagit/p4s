================================================================================
1. WHAT THIS GAME IS
================================================================================

**Permaculture 4 Squares** is a tile-based permaculture farming simulation built in
Godot 4.x. The player manages a grid farm over a **turn-based weekly cycle**:
plan tools and planting by day, spend **Energy**, queue actions, then **Sleep** to
advance the calendar and run overnight ecology (plant growth, soil exchange, weather,
spreading, harvest).

Core fantasy:
  - Soil is a living 9-stat ecosystem (moisture, nitrogen, minerals, structure,
    fungi, bacteria, macro-life, pH, toxicity) — not a single "fertility" number.
  - Plants are **active participants**: they consume gatekeeper resources, alter
    soil deltas each night, and stack vertically in **canopy / understory / ground**.
  - **Guilds** (1-tile vertical stacks) and **Superguilds** (3×3 role neighbourhoods)
    reward polyculture design.
  - Yorkshire-inspired hydrology (swales, mounds), calendar (24 fantasy months), and
    regional superguild lore inform content — even in opencore sandbox mode.

Tone: educational permaculture sim with cozy pixel presentation; commercial fork adds
ghost-story narrative (NOT in this public/opencore tree).

5. GAMEPLAY LOOP
================================================================================

**Turn** = one calendar day. Turn 1 = Dawnseed 1st, Year 1.

**Player resources (see obsidian/11_The_Weekly_Cycle.md):**
  - Energy — stamina for map tools; resets on sleep/week advance
  - Money (£) — seeds, structures (current_money in FarmDataManager)
  - Action queue — planned tile work executed during sleep animation

**Typical day:**
  1. Inspect soil (tile inspector, lenses, Hold Shift for deep tooltip)
  2. Rotovate / hoe / water / plant / build / queue earthworks
  3. Sleep → night queue runs → advance_turn() → plant biology, weather, spread

**Sleep / advance_turn (starting_map.gd):**
  - Runs queued player actions
  - Weather applies global moisture deltas (forecast-driven)
  - Per-plant biology: Phase A gatekeepers (stress/death), Phase B exchange (deltas)
  - Growth ages plants; spread; auto-harvest if enabled
  - Calendar increments; season derived from month index

**Creative mode flags** (FarmDataManager): infinite water, weatherproof, time-lapse
multiplier, zen mode — for sandbox/testing.

6. GRID CELL MODEL
================================================================================

Each cell is a Dictionary. Important keys:

LAND / TERRAIN:
  land          — "wild", "cultivated", "forest", "swale", "mound", "stream",
                  "river", "road", "bridge", "overgrown", "sand", etc.
  zone          — e.g. "polytunnel" (weather-shielded microclimate)

VERTICAL PLANT STACK (three independent layers):
  canopy        — plant id string (e.g. "apple") or empty
  understory    — plant id
  ground        — plant id
  canopy_age, understory_age, ground_age — float days since planting

V3 SOIL STATS (most clamped 0–10; pH 0–14):
  moisture, nitrogen, minerals, structure, toxicity, ph
  fungi, bacteria, macro_life, biodiversity, aeration
  depth (topsoil cm) — physical gatekeeper for taproots

STRUCTURES / FIXTURES:
  structure     — building id string (honesty_box, pig_house, …) OR legacy numeric
                  soil structure stat — code uses _cell_has_building_structure() to disambiguate

INFRA:
  seed_id       — queued seed for planting preview
  planned actions tie into action_queue / blueprints

Legacy saves with only m/n are upgraded on load to V3 via save_manager.gd.

7. SOIL ECOSYSTEM (DESIGN MODEL)
================================================================================

Documented in obsidian/00_Soil_Ecosystem.md and layer files 01–04.

Three conceptual layers for players:
  Physical  — depth, moisture, structure, temperature, sun/shade
  Chemical  — nitrogen, minerals, pH, toxicity
  Biological — fungi, bacteria, macro-life (worms/beetles)

**Gatekeepers:** Each plant has min/max moisture, nitrogen, minerals, max_toxicity,
ideal pH, min_depth, frost_hardiness, min_germination_temp, max_temp. Outside bounds
→ stress chip, zero growth, eventual death with floating text ("Thirsty!", "Root Rot!",
"Nutrient Burn!", "Toxic Soil!", etc.).

**Exchange:** Surviving plants apply per-turn deltas from CSV (see §8). Swales push
moisture to neighbours (capillary action). Mounds reduce evaporation. Weather applies
global moisture swings (+3 heavy rain, -2 dry, etc.). Polytunnel cells immune to frost.

Player-facing UI: Soil Profile inspector (soil_profile_ui.gd) with safe-range overlays
from overlapping plant requirements; ghost bars show next-turn forecast.

8a. SIX-STAGE GROWTH (VISUAL + BIOLOGY)
--------------------------------------------------------------------------------

Universal life cycle (scripts/plant_growth.gd, class_name PlantGrowth):

  Stage 0 Sown        — age == 0
  Stage 1 Seedling    — age > 0, < 33% of days_to_mature
  Stage 2 Vegetative  — 33%–66% of mature
  Stage 3 Flowering   — 66%–100% of mature
  Stage 4 Mature      — >= days_to_mature, < days_to_senescence (harvest-ready)
  Stage 5 Senescence  — >= days_to_senescence (999 = effectively perennial)

**Stage-scaled soil exchange** (IMPORTANT — as of 2026-06):
CSV deltas are BASELINE values. At runtime, nitrogen/mineral/moisture deltas multiply
by stage curve before applying to soil:

  Stage | N mult | Min mult | Moisture mult
  ------|--------|----------|---------------
  0     | 0.0    | 0.0      | 0.1
  1     | 0.25   | 0.1      | 0.5
  2     | 1.0    | 0.5      | 1.0   ← peak N hunger (leafy growth)
  3     | 0.5    | 1.0      | 1.0   ← shift to minerals
  4     | 0.2    | 1.0      | 1.0   ← fruiting mineral draw
  5     | 0.0    | 0.0      | 0.0   ← dead/dying

Positive deltas (N-fixers) scale the same way. structure_delta, toxicity_delta, and
fungal/bacterial/macro affinities are NOT stage-scaled.

UI: Hold Shift on tile → stage label + scaled next-turn exchange (plant_nutrient_forecast.gd).

**Sprites:** assets/base/sprites/flora/{plant_id}/stage_0.png … stage_5.png (200×200).
Manifest: assets/base/sprites/flora/STAGE_SPRITE_MANIFEST.txt (198 files, mostly NEED).
Fallback: legacy atlas_x until stage art exists.

11. WEATHER & FORECAST
================================================================================

Weather sources:
  - data/opencore_weather.csv for tutorial campaign
  - Procedural roll when no CSV row (sandbox)
  - data/story_weather.csv for legacy story beats

  Heavy Rain +3, Clear -0.5, Dry -2, Early Frost -0.5 + kills low frost_hardiness

WeatherDock: plain-English farmer-friendly copy; accessible detail panel.

18. HOW TO HELP AS A GAME DESIGN ASSISTANT
================================================================================

DO:
  - Think in permaculture terms: guilds, succession, soil food web, water retention.
  - Propose plant rows as CSV field sets aligned with 06_Plant_Data_Schema.md.
  - Respect the three-layer vertical stack and 1-tile vs 3×3 guild distinction.
  - Consider gatekeeper pain (too wet/dry/hungry) AND exchange deltas together.
  - Account for growth stage when discussing nutrient draw (seedling ≠ mature tree).
  - Reference Yorkshire hydrology and regional superguilds for flavour.
  - Suggest player-readable feedback (floating text, Shift tooltip, lens colors).
  - Keep opencore/premium boundary clean.

DON'T:
  - Invent a 7th growth stage or parallel fertility stat without strong justification.
  - Assume Stardew timing — turns are days; mature_turn varies 3–28+ by species.
  - Propose mechanics that require premium assets or ghost-story code in opencore.
  - Ignore spread_rate on invasives (himalayan_balsam = 10, very aggressive).
  - Treat structure cell key as only buildings — it's overloaded (see §6).

USEFUL DESIGN QUESTIONS TO EXPLORE:
  - How does a new superguild reward wetland planting (willow + reed + alder)?
  - What gatekeeper band makes Yorkshire clay feel punishing but fair?
  - Which annuals teach nitrogen bankruptcy in Gen1 tutorial?
  - How should senescence (stage 5) affect chop-and-drop vs living mulch?
  - What forecast uncertainty is fun vs frustrating at day 15?
