================================================================================
3. KNOWN / SUSPECTED COST CENTRES
================================================================================

Ask Gemini to rank and propose experiments for each:

A. PIXEL FILL / RESOLUTION
   Shipping at 2560×1600 (or native 4K) on iGPU is brutal. Stretch mode helps UI
   but still paints a huge framebuffer. Low-end path should default to 1280×720
   or 1600×900 windowed, with an explicit "Performance" display preset.

B. RENDERER / DRIVER
   Forward Plus + D3D12 may be heavier than needed for a 2D TileMap game.
   Brainstorm: Compatibility renderer? OpenGL / D3D11 fallback? Feature tags for
   a "low" export preset? Trade-offs for shaders/particles.

C. MAP SIZE × FULL-GRID PASSES
   128×128 × multiple layers × A* sync × overlay redraws.
   Ideas: smaller default creative maps; chunked/dirty-rect visuals; never call
   full update_visuals except load/wake/lens; defer overnight work across frames.

D. OVERLAYS & LENSES
   Data/energy/guild overlays that queue_redraw every frame or rebuild textures.
   Capillary shimmer, rain particles, screen shake — cheap alone, costly together
   on iGPU. Suggest "Lite visuals" toggle that disables non-essential overlays.

E. TEXTURE MEMORY
   Large flora atlas (200px cells), terrain atlas, UI. Integrated GPU shares RAM —
   thrashing looks like "slowness". Atlas size caps, compressed formats, mipmaps
   policy (nearest, often no mips for pixel art).

F. CPU / GDSCRIPT HOT PATHS
   starting_map.gd is large; sleep/overnight plant stress, pathfinding, action
   queue. Profile with Godot debugger on a mid machine, then validate on Windows
   logs. Avoid O(n²) neighbour scans without early outs.

G. AUDIO
   RadioManager / interactive music / many WAV streams. Decode + mix cost on weak
   CPUs; offer mute-by-default or "music off" for low preset.

H. COLD START
   Autoload CSV/plant data, scene tree build, first shader compile / PSO on D3D12.
   First-run hitch vs second run. Milestone timings in P4S_logs are the truth.
